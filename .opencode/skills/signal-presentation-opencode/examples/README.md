# Signal Artifact Examples

Curated examples showing how to apply the Signal philosophy and guidelines.

## north-mini-test/

This directory contains a dated historical project snapshot showing how the Signal philosophy was applied to a real multi-model evaluation harness project:

- `snapshot-metadata.yaml` — Metadata describing the snapshot's date and source commit.
- `paper.html` — The canonical two-stage research paper demonstrating serif font layout, formal headings, explicit definitions, and truth labels.
- `evidence-index.html` — The scenario evidence matrix showing tabular metrics, count-based outcome status filters, and gateway/route system columns.

## Method for Creating Custom Presentations

To build a new presentation or research exhibit:
1. Refer to the canonical guidelines and snapshotted files.
2. Apply the guidelines in `SKILL.md` (e.g., truth labels, defining load-bearing terms).
3. Do not blindly copy raw numbers or project-specific metrics. Keep your figures grounded in your project's SQLite DB or config matrices.
