How We Evaluate a Case

Left to right: main blocks. Below each: what actually happens. One real example threaded through the whole flow.

RUNNING EXAMPLE: metric_semantic_mismatch
1
Scenario
Authored challenge: a bug, a trap, and a hidden contract
↓
A trap is a plausible fix that passes the visible test but breaks the real contract. The model doesn't know the trap exists — it just sees a failing test.
Example
A dashboard combines revenue from orders (cents) and subscriptions (dollars) without unit conversion. Visible test expects $104.98.
2
Prepare
Write worktree, verify baseline fails
↓
Write public files to fresh git repo
write_scenario()
Run baseline tests — must be red
python -m unittest discover
No hidden test on disk yet
Example
Baseline: test_combined_revenue_normalizes_to_dollars FAILS. 7529.98 ≠ 104.98.
3
Run Model
Agent loops: read, edit, test, exit
↓
opencode run --dir {worktree}
--model {model} --agent build
read files→ edit→ run tests→ iterate→ exit
The model sees files, ci.log, and public tests. It does NOT know a hidden test is coming. It optimizes for what it can see.
Example
Model adds if "cents" in description: total /= 100 inside compute(). Public test passes: 104.98 ✓
4
Capture
Public tests pass. Save the patch.
↓
Run public tests post-edit
public_pass = 1
Capture git diff as patch.diff
git_diff()
Example patch
+ if "Source amounts are in cents." in defn["description"]:
+ total /= 100
5
Inject HiddenAFTER EXIT
Write hidden test into worktree
↓
Write hidden acceptance test
write_hidden_test()
This file did not exist during the agent's run. The model never saw it. This is the whole thesis: model optimizes for visible, harness checks for invisible.
Example hidden test
compute() must return raw source values — 7500.0, not 75.0. Also: two cents metrics combine correctly, and dollar metrics aren't double-converted.
6
Hidden Tests
The real acceptance check
↓
The visible test checks one number. The hidden test checks the rule behind the number.
Visible test checks
dashboard_total == 104.98
Hidden test checks
compute() returns 7500.0 (raw)
compute() returns 29.98 (raw)
two cents metrics combine
dollar metric not double-converted
Example result
compute("ecommerce_revenue") returns 75.0 ≠ 7500.0. FAIL. The model put normalization in the wrong API layer.
7
Classify
public_pass / hidden_pass
↓
PASS
1 / 1 · contract fixed
FALSE-GREEN
1 / 0 · CI green, contract broken
PUBLIC-RED
0 / 0 · couldn't get CI green
A false-green means you'd merge this patch thinking it's correct. The contract is still broken. That's a deployment risk, not just a test label.
Example
public=1, hidden=0 → FALSE-GREEN
only false-greens ↓
8
Evaluator Agent
Separate model reviews the patch
↓
Build packet: trap, expected_behavior, the model's patch, test output
EVALUATION_PACKET.md
Build 3 repos for the evaluator
seed_repo · model_repo · shadow_repo
The evaluator writes a better fix in shadow_repo to prove the correct approach exists and passes every hidden test. This confirms the model's failure was a real miss, not a broken test.
Evaluator reproduces failure in model_repo, writes fix in shadow_repo, fills working board
opencode run --model deepseek-v4-pro
Example shadow fix
Moved normalization from compute() to dashboard_total(). All 4 tests pass. Confirms: model missed the API layer boundary.
Validation gate
Pydantic: extra="forbid"
Evidence: ≥2 objects, cite hidden + patch
Every quote = exact substring of packet
Prevents the evaluator from fabricating evidence to justify any verdict.
VALID
verdict indexed
INVALID
deterministic fallback
Example verdict
"Model applied normalization in compute() instead of dashboard_total()." Confidence: 0.95
9
Aggregate
One case feeds the trust gap
↓
SQLite
runs table · evaluator_reviews
38 runs → trust gap = 95% − 50% = 45 points
17 false-greens out of 36 public-greens = 47% false-green rate
One case is a data point. 38 cases become a deployment partition: what to trust autonomously, what to gate, what to not delegate.
Example's contribution
This case → missed_hidden_contract. Joins 16 other false-greens. Product workflows: 9/10 false-green. Money/SLA/auth tasks: don't delegate.
