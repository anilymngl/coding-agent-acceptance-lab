Authoring

Scenario Authoring

Each scenario is hand-authored with a trap, a hidden contract, and metadata. The generated repo is disposable — the challenge design is the product.

trap

The tempting wrong path — a plausible minimal fix that satisfies the visible test but violates the hidden contract

expected_behavior

What a strong agent should actually do — the real contract, not just the assertion

public test

Visible to the model. Tests the obvious requirement. Like normal CI.

hidden test

Injected after agent exits. Tests edge cases, generalization, policy completeness. Never seen by the model.

Analyze
5

Analysis & Reports

Trust Metrics

  • trust_gap — public pass rate minus hidden pass rate
  • false_green_rate — false-greens as share of public-greens
  • public_red_rate — how often it can't even get CI green
  • severity_weighted_failure — hidden failures weighted by impact_weight (1-5)

Value Metrics

  • best_of_3 — 3 attempts per task, smallest passing patch wins
  • accepted_patches_per_review_hour — value delivered per engineer minute
  • median_changed_lines — patch size of accepted fixes
  • median_review_minutes — review cost estimate (manual override supported)

ci-vibe-report xray

defensible evidence pack: scorecard, control snapshot, evaluator taxonomy, claim ledger, artifact index

ci-vibe-report value

maintenance scorecard: best-of-3, reviewability table, accepted patches per hour

ci-vibe-report hidden-failures

failure inbox: per-scenario trap, patch, hidden output, short read

Strong Review

do not delegate

  • ✗ Money math (any rounding)
  • ✗ Business rules (SLAs, calendars)
  • ✗ Auth and audit correctness
  • ✗ State mutation with invariants

hidden pass <30%
