The One Question

Standard benchmarks score pass/fail. This harness scores how often you can trust a pass.

Public Tests

Visible to the model during its run — like normal CI. The model can read, run, and iterate against these.

Hidden Tests

Injected after the agent exits. Test the real contract: edge cases, generalization, policy completeness.

public_pass=1, hidden_pass=0 → false-green

The headline metric is the trust gap: public pass rate minus hidden pass rate.

The Pipeline

Eight deterministic steps. Hidden tests are never present during the model's run.

2

The Failure Shape

The Task-Type Split

The dividing line is semantic density — not model size, not task difficulty.

Reliable ≥70%

  • Mechanical migrations (logger.warn, utcnow)
  • Stale artifact regeneration
  • Import hygiene
  • Missing regression tests
  • Cursor pagination logic

Unreliable <30%

  • Money math with rounding policy
  • Business calendars & SLA rules
  • Auth and audit completeness
  • State mutation with invariants
  • Discount and billing stacking

Evidence Validation

Every quote in an evaluator review must be an exact contiguous substring of the evaluation packet.

# Pydantic schema

model_config = ConfigDict(extra="forbid")

verdict: Literal["pass", "public_green_hidden_red"]

patch_quality: int = Field(ge=1, le=5)

# harness checks quotes against packet

if quote not in packet:

  errors.append("not an exact packet substring")

Invent a quote → validation fails → falls back to deterministic summary. The evaluator cannot hallucinate its way to confidence.

The Three Zones

Where to trust it, where to gate it, where to not delegate.

Use Autonomously

merge on public pass

  • Mechanical deprecation migrations
  • Stale artifact regeneration
  • Doc/README sync
  • Missing regression tests

Gate Before Merge

hidden acceptance gate

  • Adapter & DTO normalization
  • Validation completeness
  • Pagination & cursor logic
  • Cache isolation & idempotency

Strong Review

do not delegate

  • Money math (any rounding)
  • Business rules (SLAs, calendars)
  • Auth and audit correctness
  • State mutation with invariants
