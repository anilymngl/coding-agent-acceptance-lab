Specimen 01 — The Question

Benchmarks measure whether the test passed.

We measure whether the bug was fixed.

When you reward an agent for making tests green, it optimizes for the visible assertion. It stops when CI passes. It doesn't ask "what else should be true?"

Real Fix

Public test passes. Hidden acceptance test passes. The agent understood the problem and the contract behind it.

False-Green

Public test passes. Hidden acceptance test fails. The agent satisfied the visible assertion without fixing the underlying contract. CI lied.

Public Red

The agent couldn't make CI green at all. Honest failure — at least you know it didn't work.

Specimen 02 — The Method

Visible tests during the work.
Hidden tests after the agent exits.

01

Write a broken repo

Failing test. CI log. Source code. The agent sees exactly what a real engineer would see.

02

Let the agent fix it

Same CLI, same agent loop, same tool surface for all three models. Read, edit, test, exit.

03

Run public tests

This is what every benchmark measures. This is where every benchmark stops.

04

Inject hidden tests

Edge cases. Backward compatibility. Business rules. The agent never saw these. They test whether the fix is real.

The trust gap = public pass rate − hidden pass rate. It measures how often "looks fixed" is actually "is fixed."

Specimen 06 — Deployment Policy

The model is replaceable.
The trust gate is the enduring investment.

GREEN

Merge on CI green

Mechanical work. Migrations. Fixtures. Docs. All three models: 0–14% false-green with clear specs.

AMBER

Gate on hidden acceptance

Adapters. Validators. Edge cases. Every model produced false-greens under sparse. Clear specs close the gap.

RED

Human review required

Billing. Discounts. Inventory. SLAs. 73–89% false-green under sparse. 9–33% even with clear specs.

Write a clear spec. Let the agent attempt. Run hidden acceptance. If it fails, escalate. The model is replaceable. The trust gate is the enduring investment.

CAVEATS

Gemma runs through a foreign agent loop. DeepSeek runs as a hosted API. I'm measuring system behavior, not abstract model quality. 33 scenarios. One evaluator. No external audit. First-stage evidence.
