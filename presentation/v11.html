The Trust Gap

When a coding agent makes your CI green, how often is the bug actually fixed? We tested 5 models across 3 task types with hidden acceptance tests injected after the agent exits.

The pattern holds for all of them.

North Mini · Cohere · 30B/3B Gemma 12B · Google · local DeepSeek V4 Pro · frontier API Laguna XS.2 · Poolside · 33B/3B Gemma e4b/31B · size reference

THE QUESTION

Public tests measure CI repair.
Hidden tests measure acceptance repair.

Every scenario has two layers. Public tests — visible to the agent, like normal CI. Hidden tests — injected after the agent exits. If public passes and hidden fails, CI lied.

Trust gap = public pass rate − hidden pass rate. A 30% gap means 3 out of 10 green-CI patches don't actually fix the problem.

Two prompt lanes: sparse ("CI is red, fix it") vs contract-visible (full acceptance criteria in the prompt). The gap between them tells you: is this a capability wall, or did nobody tell the model what "done" means?

The challenges were designed to create this gap. The hidden contract cannot be inferred from the public test alone. You can't deduce "half-up cent rounding" from seeing 0.29 × 3 = 87.

WHAT FALSE-GREENS LOOK LIKE

The agent satisfies the assertion. Not the rule.

decimal_money_rounding

- total = 0.0
+ total = Decimal("0")
- total += float(line["unit_price"]) * int(line["quantity"])
+ total += Decimal(line["unit_price"]) * int(line["quantity"])
return int(total * 100) ← truncates, doesn't round half-up

Test: 0.29 × 3 = 87¢. Returns 87. Green.
Hidden: 0.005 × 1 should be 1¢. Returns 0. Silent billing error.

Laguna produced this exact patch twice on independent attempts. Same wrong code, fresh start each time. Deterministic false-green.

audit_log_redaction

Test: redact password. Agent replaces top-level key. Green.
Hidden: nested metadata.api_key, profile.token, items[0].authorization. None redacted. Didn't recurse.

metric_semantic_mismatch

Dashboard combines cents + dollars. Agent normalizes in compute() — the raw API. Green.
Hidden: compute() must return raw values. Normalization belongs in dashboard_total(). Right math. Wrong layer.
