The evaluator is given a workbench, not just a diff

A false-green tells us that the agent made the visible tests pass but failed hidden acceptance. That identifies the symptom. It does not explain why the patch failed.

For selected false-greens, we give a separate evaluator agent a disposable engineering workbench.

The evaluator must work through three stages:

1. Reproduce the failure

The evaluator applies the original model patch and runs the same public and hidden tests. This checks whether the retained patch reproduces the recorded result (public tests pass, hidden tests fail). If the failure cannot be reproduced, the evidence is incomplete and the review should not be treated as validated.

2. Diagnose the missed contract

The evaluator inspects the code, test output, and authored acceptance contract. It must identify the specific mismatch between the patch and the required behavior. The diagnosis must point to concrete evidence from the workbench. A verdict without an observable code path, test failure, or contract mismatch is not enough.

3. Build a shadow fix

The evaluator then edits a clean shadow repository and attempts a different implementation. The shadow fix is tested against the same visible and hidden suites. When the shadow fix passes, it shows that the authored contract is implementable in the supplied repository and supports the evaluator’s diagnosis of a concrete gap in the original patch.

What this establishes

A validated evaluator review supports a narrow claim:

  • Under the authored benchmark contract, the original agent patch was incomplete or incorrect, and the evaluator demonstrated an alternative implementation that satisfied the same acceptance tests.

What this does not establish

The evaluator does not independently prove that:

  • the hidden tests are complete;
  • the scenario is fair or representative;
  • the benchmark contract matches every real-world requirement;
  • the evaluator’s root-cause category is uniquely correct.

The workbench makes the diagnosis executable and auditable. It does not turn one evaluator model into an external scientific authority.
